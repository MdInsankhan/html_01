SIEM Structure:

IBM Security Qradar Implementation

Introduction

This document contains the deployment architecture, prerequisites, and installation steps of IBM QRadar in the network environment. This document displays the overall Implementation Scope delivered in the IT infrastructure of United Commercial Bank Ltd.

About IBM QRadar SIEM Solution

IBM QRadar security intelligence platform provides a unified architecture for collecting, storing, analyzing, and querying log, threat, vulnerability, and risk-related data. As a result, analysts, operators & auditors using any of the modules are benefitted from the following points:

• Unified collection, aggregation, analysis of application logs, event logs, security events data & configuration files.

• A common platform for all searching, filtering, rule writing & reporting functions.

• A single user console interface for all log management, risk modeling, vulnerability prioritization, incident detection & impact analysis.

• One console security architecture approach.

Audience

This document is intended for members of the United Commercial Bank Ltd, solution architects, and technical team who have direct responsibility for the network infrastructure inside the IT network. In addition, it is advisable that those IT persons responsible for managing Network devices also review this document and provide comments and feedback where applicable.

Milestone 1: QRadar Installation

Assumptions and Dependencies by UCB

Physical Appliance

	•	Network configuration for Dell / Lenovo “iDrac / IMM” must be completed from the Effective date of this SoW.
	•	Dell / Lenovo Server “iDrac / IMM” console is accessible
	•	“iDrac / IMM” console shall be share through ‘Zoom Meeting’
	•	Raid 6 must be configured for the HDD/SDD redundancy.
	•	Only 1 virtual drive must be available after RAID configuration, if there is more than one virtual drive, Red Hat
		Enterprise Linux Server V7.7 Binary ISO will also be required to be downloaded and made available for mounting on the server.
	•	QRadar ISO file must be available on the remote system which can be mounted on the Dell / Lenovo server for installation.
	•	For non HA hosts One IP address must be available during the installation.
	•	For HA hosts three IP addresses for primary and secondary servers and virtual IP addresses must be available during the installation.
	•	Subnet mask, default gateway, FQDN, DNS Address, NTP, and SMTP addresses must be available during the installation.
	•	For non HA hosts port 443 and 22 for provided IP address should be allowed and accessible.
	•	For HA hosts port 443 and 22 for Primary, Secondary, and Virtual IP should be allowed and accessible.
	•	For non HA hosts, two interfaces will be required for the deployment, which are
	o	“iDrac / IMM” for appliance management
	o	QRadar management interface
	•	For HA hosts three interfaces will be required for the deployment, which are
	o	“iDrac / IMM” for appliance management
	o	QRadar management interface
	o	QRadar HA Sync interface

Virtual Appliance

	•	Virtual Machine must be created with the configuration provided by the IBM
	•	VMWare Center / vSphere console shall be shared through 'Zoom Meeting'
	•	QRadar ISO file must be available on the datastore and mounted on the Virtual Machine
	•	One IP address must be available during the installation.
	•	Subnet mask, default gateway, FQDN, DNS Address, NTP, and SMTP addresses must be available during the installation.
	•	Port 443 and 22 for provided IP address should be allowed and accessible.
